Implement the TicTacToe game!

For this assignment you will implement a game of TicTacToe, using the provided starter files (listed below)(it is required that you build on the files that are provided to you - do not start from scratch, an do not do a massive rewrite of what you find in the files).

Starter Files For Tic-Tac-Toe

  • HTML file
  • JavaScript file
  • Images: blank.gif   x.gif   o.gif

The basic architecture for the game is this: the page maintains 3 arrays as global variables: one for the top row of cells/blocks, one for the middle row of blocks, and one for the bottom row of blocks..  If you need to ask any questions ("Has player X won?", or "Are there any blank spaces left?") you do so by checking those arrays.  If a player puts their mark (the 'X' or 'O') in a block you do that by adjusting the array.  You will need to implement a routine which updates what the user sees on the web page based on what's in the array.  In other words: the global variables contain the "real" description of the game; any changes are made there, and then the web page is adjust to display that "real" information (as opposed to, say, basing the "real" description of the game on what's on the web page).

In order to help you break down this overall problem into smaller, more manageable parts, the following sections each focus on one (or more) functions that you will need to implement in order to successfully complete this assignment.

Part 1: Game Setup

For this assignment, open up a copy of the provided TicTacToe.html, examine it, and then open up a copy of TicTacToe.js.  These two files provide a non-trivial amount of starter code and structure; you will be doing almost all your work in the TicTacToe.js file.

 For this part of the assignment, you will need to implement the resetGame function, the renderBoard function, and the updateDisplay function, as described here:

Function: resetGame
Input: Parameters: None

User Input: None

 
Processing: This function will create all three of the arrays (one for the top row, one for the middle row, and one for the bottom row), and make sure that all the elements of the array have the value BLANK (notice that BLANK is a global variable that we'll use whenever we want to indicate that a space on the board is neither X nor O - whenever it's blank).

Additionally, it will set the game's state back to 'in progress', and update the page to indicuate who the current player is by using other functions that you will need to implement.
Output: Return Value: None


Output To User: None

 

 

Function: renderBoard
Input: Parameters: None

User Input: None

 
Processing: This function will take the global variables representing the board (i.e. the three separate arrays), and make sure that the <img> elements all display the appropriate image.  For example, the top-left image (whose id is '0_0') should display the image "Images/blank.gif" if board[0][0] is BLANK, "Images/x.gif" if board[0][0] is PLAYER_X, and "Images/o.gif" if board[0][0] is PLAYER_O.  The image next to it (id = '0_1') should display an image based on board[0][1], etc.
 
Output: Return Value: None


Output To User: The images on the page should display the appropriate picture, based on the contents of the global arrays for eac h of the three rows.

 

 

Function: updateDisplay
Input: Parameters: None

User Input: None

 
Processing: This function will put a message into the <div> element with the "currentPlayer" id telling the players whose turn it is, bsed on the currentPlayer global variable.
Output: Return Value: None


Output To User: A message indicating whose turn it is, on the page.

 

You should be able to test these functions and verify that they work. 

Part 2: Only clicking on open blocks

You should start this part of the assignment by examining the clickOnBlock function.  Most of the rest of this assignment will involve you implementing clickOnBlock's missing functions, but your overall understanding of the assignment will be greatly enhanced by understanding how each individual part works together.  So take a couple of minutes and go through clickOnBlock (and the next couple of sections), and try to develop an intuitive, holistic view of how the program will (eventually) work together.

Next, you'll need to implement and test the isBlockOccupied function, as described here:

Function: isBlockOccupied
Input: Parameters:
row: the row of the <img> that was clicked on.  0 means the top row, 1 means the middle row, and 2 means the bottom row.
col:, the column of the <img> that was clicked on.  0 means the left column, 1 means the center column, and 2 means the right column.

User Input: None

 
Processing: Determine if the block that was clicked on is an X, an O, or blank.
Output: Return Value: True if the block is currently an X or an O, false if it's blank.


Output To User: None

 

Note: In order to test this, you may want to put a bunch of alerts in the function(s) that you're writing, and make sure that it's actually doing the correct thing.
Note on the note: This is sometimes called instrumenting your program with debugging code.

Part 3: changeBlock, switchPlayer functions

For this part of the assignment, you'll need to implement and test the changeBlock and switchPlayer functions, as described below.  Additionally, you'll need to modify any other functions, as needed, in order to ensure that these get called correctly.

Function: changeBlock
Input: Parameters:
row: the row of the <img> that was clicked on.
col:, the column of the <img> that was clicked on.
toPlayer: the player that the block should be changed to.  This is either PLAYER_X or PLAYER_O

User Input: None

 
Processing: Change the appropriate slot of the board array to be either PLAYER_X or PLAYER_O, based on what toPlayer is..
Output: Return Value: None


Output To User: None.  The game will call renderBoard
 

 

Function: switchPlayers
Input: Parameters: None

User Input: None

 
Processing: Swap the current player for the other player.  If the current player is PLAYER_X, then once this function is done PLAYER_O should be the current player, and vice versa.
 
Output: Return Value: None


Output To User: None (updateDisplay will be called later to update the current player information on the page)
 

Part 4: checkForVictory

For this part of the assignment, you'll need to implement and test the routines that check and see if one of the two players has won.  There are three separate routines to implement and test - it is recommended that you implement one, then test it, then go on to the next one, etc:

Function: checkColForVictory
Input: Parameters:
col: the index of the column to check.  Remember that the first (left-most) column is column 0!
player: either PLAYER_X or PLAYER_O.  Check to see if this player has three in a row for the indicated column.

User Input: None

 
Processing: See if all three rows in the indicated column all contain player's pieces.
Output: Return Value: true if all three rows in the indicated column all contain the player's pieces; false if there's a mix, or if all three rows contain blanks.


Output To User: None - the clickOnBlock method will handle the user feedback
 


Function: checkRowForVictory
Input: Parameters:
row, which is the index of the row to check.  Row 0 is the top-most row.
player: either PLAYER_X or PLAYER_O.  Check to see if this player has three in a row for the indicated row.

User Input: None

 
Processing: See if all three columns  in the indicated row all contain player's pieces.
Output: Return Value: true if all three columns in the indicated row all contain the player's pieces; false if there's a mix, or if all three rows contain blanks.


Output To User: None - the clickOnBlock method will handle the user feedback
 

 

Function: checkDiagonalsForVictory
Input: Parameters:
player: either PLAYER_X or PLAYER_O.  Check to see if this player has three in a row for either of the two diagonals.

User Input: None

 
Processing: See if the top-left to bottom-right, or bottom-left to top-right diagonals all contain player's pieces.
Output: Return Value: true if all the blocks on either diagonal all contain the player's pieces; false if there's a mix, or if all three blocks contain blanks.


Output To User: None - the clickOnBlock method will handle the user feedback
 

Part 5: Is the game over?

The final thing to check for is the game being over with neither player winning.  In other words, if there's no open space on the board (and after we've already checked to see if one player won), then the game is over because there's no place left to move.  For this assignment, implement the following function:

Function: openSpaceExists
Input: Parameters: None

User Input: None

 
Processing: See if there's at least one element of the three global arrays that is BLANK.
Output: Return Value: true if there's at least one element of board that is BLANK; false if every single elemetn of board is either PLAYER_X or PLAYER_O.


Output To User: None
